# Slackチャンネル自動要約ツール

指定したSlackチャンネルの過去24時間のメッセージを自動で取得し、AI(LLM)に要約させて、その要約を同じSlackチャンネルに投稿し、**さらにLINEにも送信する**ツールです。

## 全体の仕組み(まずはここを理解しよう)

```
┌──────────┐  ①メッセージを取得   ┌──────────────┐
│  Slack   │ ──────────────────→ │              │
│ チャンネル │                     │ summarize.py │
│          │ ←────────────────── │ (このツール)  │
└──────────┘  ④要約を投稿         └──┬────────┬──┘
                            ②ログ送信 │        │ ⑤要約を送信
                                     ↓        ↓
                          ┌──────────────┐ ┌──────────────┐
                          │  OpenAI API  │ │   LINE API   │
                          │  (LLM/AI)    │ │ (あなたのLINE) │
                          └──────────────┘ └──────────────┘
                            ③要約文を返す
```

## 用語解説(プログラミング初心者向け)

### API とは?
「Application Programming Interface」の略で、**プログラム同士が会話するための窓口**です。
人間はSlackをアプリの画面から操作しますが、プログラムは「API」という窓口を通して
「このチャンネルのメッセージをください」「このメッセージを投稿してください」とお願いできます。

### LLM API とは?
LLM(大規模言語モデル)は、ChatGPTのような「文章を理解して文章で答えるAI」のことです。
普段はブラウザのチャット画面で使いますが、**LLM API を使うと、プログラムから自動でAIに質問して答えをもらえます**。
このツールでは OpenAI社のAPI を使い、「このメッセージログを要約して」とAIに依頼しています。

主なLLM APIには以下のようなものがあり、どれも仕組みはほぼ同じです。

| 提供元 | API名 | 代表的なモデル |
|---|---|---|
| OpenAI | OpenAI API | GPT-4o, GPT-4o-mini |
| Anthropic | Claude API | Claude Sonnet など |
| Google | Gemini API | Gemini Flash など |

このツールでは、情報が多く初心者向けの **OpenAI API(モデル: gpt-4o-mini)** を使います。

### APIキー・トークンとは?
APIを使うための「合鍵」です。誰がAPIを使ったかを識別し、悪用を防ぐために必要です。
**パスワードと同じなので、絶対に他人に見せたり、コードに直接書いたりしないでください。**
このツールでは `.env` という設定ファイルに分けて保管します。

### プロンプトとは?
AIへの「指示文」のことです。`prompt.txt` に書いてあり、
「どんな形式で」「どんなルールで」要約するかをAIに細かく指示しています。
要約の形式を変えたいときは、プログラムではなく `prompt.txt` を編集すればOKです。

### LINE Messaging API とは?
LINE公式アカウント(ボット)から、プログラムを使ってLINEメッセージを送るための窓口です。
このツールでは「プッシュメッセージ」という機能を使い、**こちらから好きなタイミングで
指定した相手(自分やグループ)に要約を届けます**。
LINEの友だち追加が必要で、送信先は「ユーザーID」または「グループID」で指定します。

## ファイル構成

| ファイル | 役割 |
|---|---|
| `summarize.py` | メインのプログラム(Slack取得→AI要約→Slack投稿→LINE送信) |
| `prompt.txt` | AIへの指示文(要約のフォーマットやルール) |
| `requirements.txt` | 必要なライブラリの一覧 |
| `.env.example` | 設定ファイルの見本(コピーして `.env` を作る) |

## セットアップ手順

### 手順1: Slackアプリ(ボット)を作る

1. [https://api.slack.com/apps](https://api.slack.com/apps) を開き、**Create New App** → **From scratch** を選ぶ
2. アプリ名(例: `要約ボット`)と、使うワークスペースを選んで作成
3. 左メニューの **OAuth & Permissions** を開き、**Scopes** の「Bot Token Scopes」に以下の4つの権限を追加する
   - `channels:history` … チャンネルのメッセージを読む権限
   - `channels:read` … チャンネルの情報(名前など)を読む権限
   - `chat:write` … メッセージを投稿する権限
   - `users:read` … ユーザー名を取得する権限
4. ページ上部の **Install to Workspace** を押してインストールを許可する
5. 表示される **Bot User OAuth Token**(`xoxb-` で始まる文字列)をコピーしておく
6. Slack上で要約したいチャンネルを開き、`/invite @要約ボット` と入力してボットをチャンネルに招待する(これを忘れるとメッセージを読めません)

### 手順2: チャンネルIDを調べる

Slackで対象チャンネルの名前を右クリック →「リンクをコピー」すると
`https://xxx.slack.com/archives/C0123456789` のようなURLが取れます。
最後の **`C` で始まる文字列** がチャンネルIDです。

### 手順3: OpenAIのAPIキーを取得する

1. [https://platform.openai.com/](https://platform.openai.com/) でアカウントを作成する
2. 支払い方法を登録し、少額(5ドル程度でOK)をチャージする
   ※ ChatGPT Plus(月額プラン)とAPIの料金は別物です。APIは使った分だけの従量課金で、
   gpt-4o-mini なら1回の要約は1円未満程度です
3. **API keys** のページで **Create new secret key** を押し、`sk-` で始まるキーをコピーする
   (キーは作成時に一度しか表示されないので注意)

### 手順4: LINEのチャネルアクセストークンと送信先IDを取得する

1. [https://developers.line.biz/console/](https://developers.line.biz/console/) にLINEアカウントでログインする
2. **プロバイダー**を作成(なければ)し、その中に **Messaging API** のチャネルを新規作成する
3. 作成したチャネルの **Messaging API設定** タブを開く
   - 一番下の **チャネルアクセストークン(長期)** の「発行」を押し、表示された長い文字列をコピーする(これが `LINE_CHANNEL_ACCESS_TOKEN`)
   - 同じ画面に表示される **QRコード** をスマホのLINEで読み取り、このボットを「友だち追加」する(これをしないとメッセージが届きません)
4. 送信先ID(`LINE_TO_ID`)を調べる
   - 自分宛てに送る場合は、自分の **ユーザーID**(`U` で始まる文字列)が必要です。
     **基本設定(Basic settings)** タブの下部にある「Your user ID」に表示されています。
   - グループに送りたい場合はグループIDが必要ですが、取得がやや複雑なので、まずは自分のユーザーIDで試すのがおすすめです。

### 手順5: 設定ファイル(.env)を作る

`.env.example` をコピーして `.env` という名前のファイルを作り、
手順1〜4で取得した値を書き込みます。

```
SLACK_BOT_TOKEN=xoxb-(手順1のトークン)
OPENAI_API_KEY=sk-(手順3のキー)
SLACK_CHANNEL_ID=C(手順2のチャンネルID)
LINE_CHANNEL_ACCESS_TOKEN=(手順4のチャネルアクセストークン)
LINE_TO_ID=U(手順4の送信先ID)
```

※ LINEの2項目を空のままにすると、Slackへの投稿だけ行い、LINE送信は自動でスキップされます。

### 手順6: ライブラリをインストールして実行する

PowerShell(ターミナル)でこのフォルダに移動して、以下を実行します。

```powershell
# 必要なライブラリをまとめてインストール(初回のみ)
# ※「pip」単体だと認識されない環境があるため「python -m pip」を使う
python -m pip install -r requirements.txt

# 実行!
python summarize.py
```

成功すると、Slackのチャンネルに要約メッセージが投稿され、LINEにも同じ要約が届きます。

## カスタマイズのヒント

- **要約の形式を変えたい** → `prompt.txt` を編集する
- **取得する期間を変えたい** → `summarize.py` の `HOURS_BACK = 24` の数字を変える
- **より賢いAIを使いたい** → `OPENAI_MODEL = "gpt-4o-mini"` を `"gpt-4o"` に変える(料金は上がります)
- **毎朝自動で実行したい** → Windowsの「タスクスケジューラ」に `python summarize.py` を登録する

## よくあるエラーと対処法

| エラー | 原因と対処 |
|---|---|
| `not_in_channel` | ボットがチャンネルに招待されていない。`/invite @ボット名` を実行する |
| `missing_scope` | 手順1-3の権限が足りない。権限を追加したら**再インストール**が必要 |
| `invalid_auth` | トークンの値が間違っている。`.env` を確認する |
| `AuthenticationError`(OpenAI) | APIキーが間違っているか、チャージ残高がない |
| `KeyError: 'SLACK_BOT_TOKEN'` | `.env` ファイルがないか、名前が間違っている(`.env.txt` になっていないか確認) |
| LINE送信が失敗(コード: 401) | `LINE_CHANNEL_ACCESS_TOKEN` が間違っている。再発行して `.env` を更新する |
| LINE送信が失敗(コード: 400) | `LINE_TO_ID` が間違っているか、ボットを友だち追加していない |
| LINE送信をスキップ | `.env` のLINE2項目が空。値を入れると送信されるようになる |
